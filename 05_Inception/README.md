# Inception

This project from 42 school by jumarque aims to broaden your knowledge of system administration by using Docker. In this tutorial You will virtualize several Docker images, creating them in your new personal virtual machine. In this README.md you will have an inception tutorial to know how the project works.

---

## 1. Introduction & Roadmap

The goal of Inception is to build a fully functional infrastructure using **Docker Compose**. You won't just pull images from DockerHub; you will build them from scratch using **Dockerfiles** based on Debian (Penultimate Stable Version).

### Important things to read before beginning
1.  **Don't try to do all the containers at once:** You will get lost. Follow a modular approach.
2.  **Step 1: Nginx:** Start by displaying a simple `index.html`.
    * Learn to launch and execute an image **without docker-compose** first.
    * Test HTTP (80) and then move to HTTPS (443) with SSL.
3.  **Step 2: WordPress:** Use PHP-FPM to process dynamic content.
4.  **Step 3: MariaDB:** Finalize by connecting your database to WordPress.

## 2. Definitions

### Core Concepts
* **Docker:** An open platform that separates applications from infrastructure. It allows you to package an application with all its dependencies into a standardized unit called a **container**.
* **Dockerfile:** A text document containing all the commands (FROM, RUN, COPY, etc.) needed to assemble a Docker image.
* **Docker Compose:** A tool for defining and running multi-container applications. It uses a `docker-compose.yml` file to manage services, networks, and volumes as a single stack.
* **Docker Volume:** A mechanism for persisting data generated by and used by Docker containers. Unlike Bind Mounts, volumes are completely managed by Docker.
* **Docker Network:** A virtual network that allows containers to communicate with each other using their service names as hostnames.

## 3. Infrastructure (The "Mandatory" Stack)

### Nginx (The Gateway)
Nginx acts as the entry point (Reverse Proxy). It handles TLS/SSL encryption and directs traffic to the appropriate service.
* **Key Config:** Use `listen 443 ssl`, define `server_name`, and configure `ssl_protocols TLSv1.2 TLSv1.3`.
* **FastCGI:** Nginx doesn't "speak" PHP. It uses `fastcgi_pass` to send requests to the WordPress container on port 9000.

### WordPress (The Application)
WordPress runs via **PHP-FPM** (FastCGI Process Manager).
* **Setup:** Requires `wp-cli` to automate installation, database creation, and user management without a GUI.
* **Persistence:** The `/var/www/html` directory must be mounted to a volume to keep your site's data.

### MariaDB (The Database)
The relational database where all WordPress posts, users, and settings are stored.
* **Security:** Requires a root password and a specific user for WordPress.
* **Initialization:** Uses `.sh` or `.sql` scripts to create the database schema upon the first boot.

## 4. Bonus Services

### Redis (Cache)
An in-memory data structure store used as a database cache to speed up WordPress page loading.
* **Integration:** Requires the `redis-cache` plugin in WordPress.

### FTP Server (File Access)
Uses **vsftpd** (Very Secure FTP Daemon) to allow users to upload files directly to the WordPress volume.
* **Passive Mode:** Requires a specific range of ports to be opened in the docker-compose.

### Adminer (DB Management)
A lightweight alternative to phpMyAdmin. A single PHP file that allows you to manage your MariaDB database through a web interface.
* **URL:** Usually accessed via `https://jumarque.42.fr/adminer`.

### Static Site (Monitoring)
A separate small website (HTML/JS) served on a subdomain (`static.jumarque.42.fr`) to monitor service health.

### Ollama (AI Service)
A local Large Language Model (LLM) runner. It allows you to run models like Llama 3 or Mistral inside your infrastructure.
* **API:** Listens on port 11434.

### Pro-Tip: Cleaning the Host
Before launching your project, ensure no local services are hogging the ports:
```bash
sudo service nginx stop
sudo service apache2 stop
sudo service mysql stop
```
This ensures your Docker containers have full access to ports 80, 443, and 3306.
